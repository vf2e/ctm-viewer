# QML 公共组件使用规范

适用范围：`ui/Pages2/Qml/components/` 下新增或重构的抽屉页面（用户、历史、设置、方案）。

视觉与交互标准见同目录 **[软件设计风格规范.md](./软件设计风格规范.md)**（色彩、字号、间距、弹窗、禁止事项等）。

## 1. 统一原则

- 页面业务逻辑可以不同，视觉基础层必须统一。
- 在抽屉模块中，默认不直接写原生 `Button` / `TextField` / `TabButton`。
- 优先使用 `common` 目录下组件：
  - `FlatButton.qml`
  - `FlatTextField.qml`
  - `FlatDatePicker.qml`（日期选择，样式与 FlatTextField 一致）
  - `FlatTabButton.qml`
  - `FlatCard.qml`
  - `FlatProgressBar.qml`（水平进度条，内嵌圆角填充与动画）
  - `FlatConfirmDialog.qml`（二次确认弹窗，样式与 `FlatMessagePopup` 一致：400 宽、24/20 边距、44 图标、底部分割线 + 取消/确认）
  - `FlatMessagePopup.qml`（操作结果提示弹窗，成功/失败/警告/信息，挂 `main2` 的 `messagePopup`；同 `FlatConfirmDialog` 视觉规格，底部「知道了」）

## 2. 组件选型规则

- `FlatButton`
  - `tone: "primary"`：主操作（查询、保存、绑定、导出、开始等）
  - `tone: "secondary"`：次操作（关闭、取消、刷新、翻页等）
  - `tone: "danger"`：危险操作（删除、停止等）
- `FlatTextField`
  - 所有单行输入统一使用该组件，减少重复 `background/border` 代码。
- `FlatDatePicker`
  - 所有日期选择（出生日期、查询起止日期等）统一使用该组件，勿直接使用 `YDatePicker`。
- `FlatTabButton`
  - 所有 `TabBar` 的按钮统一使用该组件。
  - 必须绑定 `selected: tabBar.currentIndex === index`。
- `FlatCard`
  - 用于可复用卡片容器。
  - 需要阴影时设置 `useShadow: true`，普通内容区可关闭阴影。

## 3. 代码模板

```qml
import "../common"

FlatButton {
    text: "保存"
    tone: "primary"
    implicitHeight: ctlHeight
}
```

```qml
TabBar {
    id: modeTab
    FlatTabButton { text: "标准"; selected: modeTab.currentIndex === 0 }
    FlatTabButton { text: "脉冲串"; selected: modeTab.currentIndex === 1 }
}
```

## 4. 页面实现约束

- 抽屉页面内新增交互控件时，先判断是否能由 `Flat*` 组件覆盖。
- 如需局部特殊样式，优先通过组件属性扩展，不要在业务页复制大段样式。
- 页面级颜色、圆角、边框保持与当前抽屉风格一致（扁平化、浅色层级）。

## 5. 自检清单（提交前）

- 是否存在原生 `Button {` / `TextField {` / `TabButton {`（抽屉页面默认不允许）。
- 主次危险按钮语义是否正确（`primary/secondary/danger`）。
- Tab 的选中态是否通过 `selected` 显式绑定。
- 新增样式是否可下沉到 `common`（避免重复）。

## 6. 扩展建议

- 操作结果提示统一使用 `windowRoot2.messagePopup.showSuccess()` / `showError()`，不要在业务页内再写 `Popup`。
- 如果出现第三次以上重复样式，必须提取到 `common` 新组件。
- 新增组件命名遵循 `FlatXxx.qml`，并同步加入 `res.qrc`。
